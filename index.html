  • Added NaN guard: showResults now safely defaults to 0 if score is missing or malformed, preventing rendering errors.

Previous version
  • Auto-clear analyze tab: URL field and results are automatically cleared when navigating away.
  • Removed demo chips: Cleaned up the analyze tab by removing the demo scenario buttons.
v1.4.0
Previous version
  • Clickable history: Every entry in Post History is now clickable — opens the full analysis view in the Analyze tab exactly as it appeared when first run.
  • Narrative framing: App reframed from "manipulation detection" to "narrative alignment" — the question is now whose agenda a post serves and what it leaves out, not whether it's a lie.
  • Missing context field: Each entity match now includes a "What this post leaves out" section — the facts and counter-narrative the entity would not want the reader to see.
  • Alignment color spectrum: Replaced red/amber/green danger colors with a cool-to-warm intensity spectrum: muted blue (weak) → blue-purple (moderate) → amber-orange (strong) → deep red-orange (very strong). Color reflects alignment intensity, not danger.
  • Overall dial reframed: Dial now shows strongest entity alignment score and names the entity, not a "manipulation probability."
  • Updated server prompt: Claude now asked to identify missing context per match, not just explain why a post aligns.
v1.3.0
Previous version
  • Proxy server: Post text is now fetched automatically from X, Facebook, and Instagram — no more manual copy-pasting. Requires deploying the companion whos-behind-that-server to Render.
  • API keys moved to server: Your Anthropic key is now stored securely as a server environment variable — never exposed in the browser. Users just paste a URL and get results.
  • Server URL field: Replaces the API key fields in the UI. Paste your Render server URL once and it's saved.
  • Graceful fallback: If automatic fetching fails (private post, deleted, login-gated), the app reveals a manual text paste field so analysis can still proceed.
  • One-shot analysis: A single server call now handles both fetching and scoring, reducing latency.
v1.2.0
Previous version
  • Reweighted scoring engine: Combined score now weights hidden interest at 55%, modus operandi at 35%, and public narrative at only 10% — reflecting that manipulation is primarily detected through who benefits and how it was constructed, not just what it says.
  • Richer Claude prompt: The scoring prompt now explicitly instructs Claude on the philosophy of manipulation detection, with detailed guidance on scoring each dimension independently and citing specific post phrases as evidence.
  • Updated UI labels: Dimension bars in entity match cards now show their weights (55% / 35% / 10%) so the scoring logic is transparent to the user.
  • Improved "why" explanations: Claude now returns 2–3 sentence explanations per match, specifying which hidden interest is served and which MO tactics are present.
  • Updated demo scores: All three demo scenarios recalculated under the new weighting model.

How to deploy

This app has two parts: the frontend (this HTML file → GitHub Pages) and the server (Node.js → Render). Both are free to host.
Part 1 — Frontend (GitHub Pages)
1

Create a GitHub repo

Name it whos-behind-that. Set to Public. Add a README.

2

Upload index.html

Upload this file renamed to index.html. Commit to main.

3

Enable GitHub Pages

Settings → Pages → Deploy from branch → main / (root). Your site: https://YOUR-USERNAME.github.io/whos-behind-that/

Part 2 — Server (Render)
4

Create a second GitHub repo

Name it whos-behind-that-server. Upload the three server files: server.js, package.json, .gitignore.

5

Deploy to Render

Go to render.com → sign up with GitHub → New → Web Service → connect your server repo. Settings: Build command npm install, Start command npm start, Instance type Free.

6

Add your Anthropic API key

In Render, go to your service → Environment → Add variable: ANTHROPIC_API_KEY = your sk-ant-... key. Click Save — Render will redeploy automatically.

7

Connect the app to your server

Copy your Render service URL (e.g. https://whos-behind-that-server.onrender.com). Paste it into the server URL field at the top of the Analyze tab. It turns green when connected.

8

Share the app

Send anyone your GitHub Pages URL. They just paste a post URL and click Analyze — no setup, no API keys needed on their end.
