How to deploy

This app has two parts: the frontend (this HTML file → GitHub Pages) and the server (Node.js → Render). Both are free to host.
Part 1 — Frontend (GitHub Pages)
1

Create a GitHub repo

Name it whos-behind-that. Set to Public. Add a README.

2

Upload index.html

Upload this file renamed to index.html. Commit to main.

3

Enable GitHub Pages

Settings → Pages → Deploy from branch → main / (root). Your site: https://YOUR-USERNAME.github.io/whos-behind-that/

Part 2 — Server (Render)
4

Create a second GitHub repo

Name it whos-behind-that-server. Upload the three server files: server.js, package.json, .gitignore.

5

Deploy to Render

Go to render.com → sign up with GitHub → New → Web Service → connect your server repo. Settings: Build command npm install, Start command npm start, Instance type Free.

6

Add your Anthropic API key

In Render, go to your service → Environment → Add variable: ANTHROPIC_API_KEY = your sk-ant-... key. Click Save — Render will redeploy automatically.

7

Connect the app to your server

Copy your Render service URL (e.g. https://whos-behind-that-server.onrender.com). Paste it into the server URL field at the top of the Analyze tab. It turns green when connected.

8

Share the app

Send anyone your GitHub Pages URL. They just paste a post URL and click Analyze — no setup, no API keys needed on their end.

Part 3 — Shared history (PostgreSQL)
9

Create a PostgreSQL database on Render

In your Render dashboard, click New → PostgreSQL. Name it whos-behind-that-db. Select the Free plan. Click Create Database.

10

Connect the database to your server

In your PostgreSQL instance, go to Connections and copy the Internal Database URL. Go to your whos-behind-that-server web service → Environment → Add variable: DATABASE_URL = the URL you just copied. Save — Render will redeploy.

11

Update package.json

Upload the new package.json to your whos-behind-that-server GitHub repo — it now includes the pg database driver. The table is created automatically on first server start.

12

Verify

Visit your Render server URL in a browser. You should see "db": true in the response JSON, confirming the database is connected. All scans will now be shared across all users.
